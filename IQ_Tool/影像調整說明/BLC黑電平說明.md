# BLC 黑電平校正（Black Level Correction）

## 1. 定義
BLC（Black Level Correction，黑電平校正）是 ISP 流程中的前端校正步驟，用來扣除 sensor 在無光或低光時仍會輸出的底部偏移值，使 RAW 的黑位回到接近 0。[web:1][web:2]

這個偏移通常來自暗電流、器件偏置、ADC pedestal，以及讀出電路的固定誤差。[web:1][web:13]

---

## 2. 為什麼要做
若不做 BLC，畫面暗部會發灰、黑位抬高，甚至出現偏綠或偏紫等色偏問題。[web:3][web:10]

BLC 的目的包括：
- 消除 sensor 固有偏移。[web:1][web:2]
- 讓 R、Gr、Gb、B 四個通道黑位一致。[web:3][web:15]
- 提供正確的 RAW 基準，避免影響 AWB、Demosaic、Gamma 與後續影像處理。[web:1][web:3]

---

## 3. 在 ISP 流程中的位置
BLC 通常位於 ISP 最前面，在 RAW 進入白平衡、去馬賽克、降噪、色彩校正之前先執行。[web:1][web:2]

```text
Sensor RAW -> BLC -> AWB -> Demosaic -> NR -> Gamma -> Color Correction
```

---

## 4. 基本原理
sensor 在完全無光時，理想輸出應為 0，但實際上會有一個非零黑電平值 \(B\)。  
因此需要把原始像素值 \(P_{raw}\) 扣除該偏移：

\[
P_{blc} = \max(P_{raw} - B, 0)
\]

若是 Bayer RAW，通常會對四個通道分別處理：

\[
P'_{c}(x,y) = \max(P_{c}(x,y) - B_{c}, 0)
\]

其中：
- \(c \in \{R, Gr, Gb, B\}\)
- \(B_c\) 為各通道黑電平值

若還需要避免超出範圍，也可再加上飽和限制：

\[
P'_{c}(x,y) = \min(\max(P_{c}(x,y) - B_{c}, 0), P_{max})
\]

---

## 5. 校正資料來源
常見黑電平來源有兩種：

### 5.1 Optical Black（OB）區
sensor 會預留一部分不受光的像素區域，用來估計黑電平。[web:5][web:13]

### 5.2 黑場 RAW
將鏡頭遮光或拍攝全黑畫面，再統計 RAW 值作為校正依據。[web:5][web:15]

---

## 6. 詳細校正步驟

### Step 1：取得黑場或 OB 資料
拍攝遮光 RAW，或讀取 sensor 的 OB 區資料。[web:5][web:13]

### Step 2：分離四個 Bayer 通道
把 RAW 拆成：
- R
- Gr
- Gb
- B

因為不同通道的黑電平可能不同。[web:3][web:15]

### Step 3：統計黑電平
常見統計方法：
- 平均值。
- 中位數。
- 去除異常值後再平均。[web:4][web:15]

例如：

\[
B_c = \text{mean}(P_{c,1}, P_{c,2}, \dots, P_{c,n})
\]

或：

\[
B_c = \text{median}(P_{c,1}, P_{c,2}, \dots, P_{c,n})
\]

### Step 4：執行扣除
對每個像素扣除對應通道黑電平：

\[
P'_{c} = P_{c} - B_{c}
\]

若結果小於 0，則截斷為 0。[web:1][web:3]

### Step 5：必要時做線性拉伸
部分平台在扣除黑電平後，會對 G 通道做線性拉伸以維持亮度範圍：

\[
G_{out} = G_{in} \times \frac{255}{255 - BL}
\]

這樣可以減少扣黑後的整體亮度壓縮。[web:5][web:13]

### Step 6：驗證效果
檢查：
- 黑場是否接近 0。
- 暗部是否保有細節。
- 是否有色偏。
- 高 ISO 是否穩定。[web:1][web:10]

---

## 7. 常見校正方法

### 7.1 固定值法
對每個通道直接扣除固定黑電平值。[web:3][web:15]

適用情境：
- sensor 特性穩定。
- 低 ISO 或固定工作條件。

### 7.2 ISO 聯動法
不同 ISO 分別量測黑電平，建立 LUT，運行時依 ISO 查表或插值。[web:5][web:15]

適用情境：
- 黑電平會隨增益變化。
- 需要更穩定的暗部表現。

### 7.3 曲線 / 局部法
對黑場不同區域或不同條件建立更細的補償模型。[web:1][web:5]

適用情境：
- 黑電平有空間不均勻。
- 高階 sensor 或高要求畫質調校。

---

## 8. 實務調校重點
- 黑電平不能扣太多，否則會把暗部細節一起吃掉。[web:5][web:13]
- R、Gr、Gb、B 四個通道最好分開量測，不要混成單一值。[web:3][web:15]
- 高 ISO、長曝光、溫度上升時，黑電平可能變動，要重新標定。[web:1][web:13]
- 若畫面偏綠或偏紫，常見原因之一就是各通道黑位不一致。[web:10][web:13]

---

## 9. 常見錯誤
- 直接用彩色場景估黑電平。
- 所有通道共用同一個黑值。
- 忽略 ISO / 溫度變化。
- 扣完後沒有做 0 截斷。
- 只看黑場，不看實際成像效果。[web:4][web:15]

---

## 10. 範例
假設黑場量到：
- \(B_R = 64\)
- \(B_{Gr} = 63\)
- \(B_{Gb} = 62\)
- \(B_B = 65\)

某像素原始值：
- R = 120
- Gr = 118
- Gb = 117
- B = 125

校正後：

\[
R' = 120 - 64 = 56
\]
\[
Gr' = 118 - 63 = 55
\]
\[
Gb' = 117 - 62 = 55
\]
\[
B' = 125 - 65 = 60
\]

若某值小於 0，則直接設為 0。

---

## 11. 一句話總結
BLC 的核心就是：先量出 sensor 的黑位偏移，再在 RAW 階段按通道扣除，讓真正的黑回到基準點。[web:1][web:2]
