# 灰階卡 Q‑13 — 業界判定影像標準（技術報告）

## 摘要
本報告針對業界常用的「Q‑13 灰階卡 / 色階卡」作技術說明與實務導引，包含：Q‑13 的物理規格與量測參數、在影像擷取（掃描/數位拍攝）中如何作為色彩與調性（tone）判定標準、配合業界品質規範（FADGI、Metamorfoze、ISO 19264）時的檢測流程與接受準則，以及操作上常見問題與 QC（品質把關）檢核表。

目標讀者：數位典藏工程師、攝影/掃描操作者、影像測試與品質管理人員。

---

## 1. Q‑13 是什麼（背景）
Q‑13（Kodak/Tiffen Q‑13 Color Separation Guide & Grayscales）是一組業界長期使用的色階/灰階參考卡組。它包含色階（color patches）與灰階（20 階 neutral gray steps），設計用來：校正白平衡、確定曝光與黑/白點、繪製 OECF（Opto‑Electronic Conversion Function）/tonescale、以及作為色彩及調性一致性檢測的標準目標。

用途重點：
- 量測調性（tonescale）及建立理想的 RGB aim‑points（依 patch 的反射率計算）。
- 作為每張拍攝包含的校正參考（在每張檔案或樣本拍攝中放入或在測試場景中使用）。

---

## 2. Q‑13 物理規格與代表性參數（摘要）
- 灰階 patch 數：20 個（編號 0 至 19），加上理想 white point 與 black point 參考。
- 常見尺寸（小版）：約 6×20 cm；也有 8"（約 21 cm）標準版。
- 每個 patch 有 Kodak 發表的 nominal density/reflectance 值，可用來計算理想的 RGB aimpoints（依 capture gamma 決定）。

## 3. 業界標準、工具與量測方法（概覽）
- **FADGI（Federal Agencies Digital Guidelines Initiative）**：建議在每次擷取時包含灰階與色彩參考，並以量化的 aimpoints/tolerances（星等系統 1~4★）檢測 OECF、白平衡、ΔE 色差、照度均勻性、雜訊等指標。
- **Metamorfoze（荷蘭）**：保存級指南，注重整個灰階的正確曝光與 tonescale；在新版中已引入更嚴謹的測量（例如使用 Munsell 線性灰階作為技術目標）。
- **ISO 19264‑1 / ISO 19263 系列**：提供專業影像系統品質評析方法（解析度、色差、noise、OECF 等）。

常用軟體/工具：OpenDICE / DICE / iQ‑Analyzer / UTT / 自製測量腳本（讀取每 patch 的 ROI，輸出 L*a*b*、ΔE2000、OECF 曲線等）。

## 4. 關鍵量測項目與接受準則（實務化）
> 以下為常見且可量測的 QC 指標（以館藏保存級/典藏需求為例）：

1. **白平衡（colorimetric white balance）**
   - 以灰階中性 patch 的 a*、b*（Lab）偏差衡量；FADGI/Metamorfoze 建議使用 ΔE(a*b*) 或 ΔE2000 指標來量化。高品質目標通常要求平均 ΔE2000 小於 3（若專案需求更高則朝 <2 甚至 <1）。

2. **色差（Color Accuracy）**
   - 使用色階或色卡測 ΔE2000（與目標 Lab 值比對）；FADGI 之星等代表不同容忍度（4★ 最嚴格）。

3. **調性/tonescale 與 OECF（Opto‑Electronic Conversion Function）**
   - 檢查 R/G/B 與 luminance 的 OECF 是否落在目標曲線範圍；可選 γ=2.2（常用）或根據專案規範選擇。若偏離，需調整曝光或 capture gamma。

4. **Gain Modulation（增益調變）**
   - 衡量局部 OECF 斜率（中高亮區）；Metamorfoze 與其他指南通常要求 highlight 範圍的 gain modulation 在 **約 80%–110%** 之間（過低或過高代表細節被壓縮或過度拉伸）。

5. **照度非均勻性（Illuminance Non‑uniformity / Shading）**
   - 畫面中心與邊緣亮度差異必須在可接受範圍（依 ISO / FADGI 量化），否則需改善光源排列或使用漫射設備。

6. **動態範圍 / Dmax‑Dmin**
   - 以 Q‑13 的黑白點作為參考確認擷取系統是否能覆蓋原稿的可見 tonal 範圍。

7. **噪訊與 SNR（Signal‑to‑Noise Ratio）**
   - 在暗階 patch 上檢查雜訊水準，確保 SNR 足以保存原始細節。

## 5. 推薦的拍攝 / 校正流程（步驟化）
1. **準備**：將 Q‑13 目標與被攝物置於同一平面，確保目標不在主體陰影下；若原稿較大無法放入同張，則至少在同光源、同設定下拍攝一張包含 Q‑13 的參考影像。
2. **光源**：採用接近標準視覺照明的光源（建議 D50 / 5000 K（視覺檢視用）或依專案規範選擇），並確保 CRI 高且照度均勻。
3. **相機/掃描設定**：RAW 格式、關閉自動白平衡與自動增益；固定光圈/快門以取得理想曝光（使 Q‑13 的 A/M/B 三點落在可量測範圍）。
4. **量測**：在 RAW 檔（或無色彩管理的位元資料）上取 ROI（平均）讀取每個灰階 patch 的 RGB 值，轉換成 Lab（需資訊關鍵：使用同一色彩空間與白點），計算 OECF 與 ΔE2000。
5. **調整**：使用 capture 軟體調整 black/white/midpoint（或套用 ICC in‑situ profile）使 RGB aimpoints 與理想值接近；重拍並再次量測直到達到可接受容差。
6. **記錄**：存檔時記錄測試報告（包含照明條件、相機參數、測量結果、軟體版本與校正紀錄），以便日後追溯與品質保證。

## 6. QC 檢核表（快速檢查）
- [ ] Q‑13 有無放置於同一平面且無陰影？
- [ ] 光源色溫（D50/5000K 或專案指定）與均勻性符合規範？
- [ ] 拍攝使用 RAW，關閉自動白平衡/自動增益？
- [ ] A（paper white）、M（mid gray）、B（near black）三點位於合理數值範圍？
- [ ] OECF 曲線（R/G/B）落在專案允收範圍？
- [ ] Gain modulation（highlights）介於 80%–110%（或專案指定）？
- [ ] ΔE2000（色差）平均值與最大值是否低於專案容忍度（建議：平均 <3 為較佳）？
- [ ] 照度非均勻性、邊緣暗角是否在可接受範圍？

## 7. 常見問題與對策（FAQ）
**Q：Q‑13 與其他色卡（例如 ColorChecker / IT8）怎麼選？**
A：Q‑13 是以灰階/tonescale 為主的目標；若要同時量測色彩準確性，建議同時使用色彩目標（ColorChecker SG / IT8）。

**Q：新品 Q‑13 與舊版是否需要重新量測？**
A：任何參考卡都會隨時間與使用/清潔狀況改變（漂白、汙染）。定期（視使用頻率）用分光儀或由供應商提供的 Lab 檔比對校正資料，若偏離需替換或重新建立目標參數。

## 8. 附錄：數值與計算（簡要）
1. **Reflectance → 理想 channel value（8‑bit 下）**
   - ChannelValue = 255 × (Reflectance^(1/γ))，常用 γ=2.2（或依專案決定）。
2. **建議 ROI 讀取方式**
   - 每個 patch 取中心稜形區域（避開邊緣與印刷/裁切瑕疵），使用平均值（多像素）以降低量測雜訊影響。

---

## 9. 結論與建議
Q‑13 是一個穩定且業界通用的灰階/色階參考工具，能有效支援影像擷取時的曝光、白平衡與 tonescale 校正。為符合保存級品質（依 FADGI / Metamorfoze / ISO 類別），建議結合：嚴謹的光源控制（D50）、RAW 擷取、量化檢測流程（OECF、ΔE、gain modulation）以及良好的記錄防護流程。
